# Skill: review

Evaluate a draft and produce structured, actionable feedback.

## Input

- A draft document
- Optional: review criteria, target audience, intended purpose

## Output

A review report with the following sections.

## Review report structure

### Summary verdict
One paragraph: overall quality assessment and the single most important thing to fix.

### Strengths
Bullet list — what is working well and should be preserved.

### Issues

Each issue must include:
- **Severity**: `critical` | `major` | `minor`
- **Location**: section or quote
- **Problem**: what is wrong and why
- **Suggestion**: concrete fix

### Checklist
Quick pass on common criteria:

| Criterion | Pass | Notes |
|-----------|------|-------|
| Clear central argument | | |
| Logical flow between sections | | |
| Concrete examples used | | |
| Appropriate length | | |
| Consistent tone | | |
| No unexplained jargon | | |

## Instructions

- Be direct and specific. Vague praise or criticism is not useful.
- Prioritize critical issues; do not bury them in minor feedback.
- Suggest fixes, not just problems.
- Respect the author's voice — flag style issues only if they harm clarity.
